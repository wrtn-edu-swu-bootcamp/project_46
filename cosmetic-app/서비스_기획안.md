## 프로젝트 기획 의도

### 배경
화장품 성분에 대한 정보는 넘쳐나지만, 비전공자나 일반 소비자가 이해하기 어려운 전문 용어로 가득합니다. 후기나 인터넷 검색으로는 "내 피부 타입과 비슷한 사람들에게 맞는지" 정도만 파악할 수 있을 뿐, 정작 중요한 **성분의 작용 원리**나 **다른 성분과의 상호작용**은 알기 어렵습니다.

### 문제 인식
- 화장품 성분 정보의 접근성 부족
- 피부 타입별 맞춤 정보 부재
- 성분 간 상호작용(좋은 조합/피해야 할 조합) 정보 부족
- 전문 용어로 인한 이해의 어려움

### 해결 방안
더마북은 식약처 공공데이터와 AI 기술을 활용하여 화장품 성분의 **작용 과정**을 쉽게 이해할 수 있도록 돕습니다. 단지 "좋다/나쁘다" 보다는, **왜 좋은지, 어떻게 작용하는지, 내 피부에는 어떤 영향을 미치는지**를 명확히 알려줌으로써 사용자가 자신의 피부를 더 잘 이해하고 현명한 선택을 할 수 있도록 지원합니다.

## 주요 기능

### 1. 성분 검색
- **로컬 데이터베이스**: 엄선된 30+ 화장품 성분 정보
- **식약처 공공데이터 연동**: 2만+ 성분 실시간 검색
- 성분별 상세 정보 (효능, 작용 메커니즘, 적합한 피부 타입, 조합 정보)
- 즐겨찾기 기능

### 2. AI 뷰티 상담
- Groq AI 기반 실시간 뷰티 상담
- 사용자 피부 타입 기반 맞춤 조언
- 화장법, 스킨케어 루틴, 성분 추천

### 3. 피부 타입별 맞춤 추천
- 온보딩 시 피부 타입, 나이대, 피부 고민 설정
- 홈 화면에서 피부 타입별 성분 추천
- AI 상담 시 자동으로 피부 정보 반영

### 4. 화장법 팁
- 피부 타입별 화장법 가이드
- 카테고리별 분류 (베이스, 색조, 스킨케어)

### 5. 마이페이지
- 프로필 관리 (피부 타입, 고민 수정)
- 즐겨찾기 관리
- 피해야 할 성분 관리
- 다크모드 지원

## 기술 스택

### Frontend
- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **Animation**: Framer Motion
- **Icons**: Lucide React

### Backend/API
- **API Routes**: Next.js API Routes
- **AI**: Groq API (Llama 3.3 70B)
- **Public Data**: 식품의약품안전처 화장품 원료성분정보 API

### Deployment
- **Hosting**: Vercel
- **Version Control**: Git/GitHub

## 데이터 출처

1. **식품의약품안전처 화장품 원료성분정보**
   - API Key: 공공데이터포털 발급
   - 2만+ 화장품 성분 데이터

2. **자체 큐레이션 데이터**
   - 30+ 주요 성분 상세 정보

## 사용자 플로우

1. **첫 방문**
   - 온보딩 화면 (웰컴 → 프로필 설정)
   - 닉네임, 나이대, 피부 타입, 피부 고민 입력

2. **홈 화면**
   - 피부 타입별 맞춤 성분 추천
   - 오늘의 팁
   - 빠른 검색 접근

3. **성분 검색**
   - 로컬 DB 검색
   - 식약처 DB 검색
   - AI 효능 검색

4. **AI 상담**
   - 실시간 채팅 형식
   - 피부 타입 기반 맞춤 답변

5. **마이페이지**
   - 프로필 관리
   - 즐겨찾기/피해야 할 성분 관리
